Cheat Sheet Git

Resumen de control de versiones, ramas, repositorios remotos, inspección del historial, recuperación y estrategias seguras de merge.

Conceptos clave

Repositorio Commit Branch Merge
  • Working tree: archivos que estás editando.
  • Staging area: cambios preparados con git add.
  • Commit: instantánea del proyecto.
  • Branch: línea de desarrollo independiente.
  • Remote: repositorio remoto, por ejemplo origin.

Estados más comunes

  • untracked: archivo nuevo no seguido por Git.
  • modified: archivo modificado.
  • staged: cambio listo para commit.
  • committed: cambio ya guardado en el historial.
  • ahead/behind: tu rama está adelantada o atrasada respecto al remoto.
  • detached HEAD: estás parado en un commit, no en una rama.
Flujo mental básico: editar archivos → git status → git add → git commit → git push. Para integrar trabajo ajeno: git fetch, revisar, y luego merge o pull.

8. Merge: conceptos, comandos y uso seguro

ComandoUsoSintaxisEjemplo
git merge ramaFusiona una rama dentro de la rama actual.git merge ramagit merge main
git merge --no-ff ramaFuerza un commit de merge aunque Git pudiera avanzar en línea recta.git merge --no-ff ramagit merge --no-ff feature-login
git merge --ff-only ramaSolo permite fast-forward; si no se puede, falla.git merge --ff-only ramagit merge --ff-only main
git merge --abortCancela un merge en conflicto y vuelve al estado anterior.git merge --abortgit merge --abort
git merge --continueContinúa un merge después de resolver conflictos.git merge --continuegit merge --continue
git mergetoolAbre una herramienta gráfica o configurada para resolver conflictos.git mergetoolgit mergetool
Regla de oro del merge: Git fusiona la rama que nombras dentro de la rama en la que estás parado. Si estás en fran y ejecutas git merge main, estás trayendo main a fran. Si estás en main y ejecutas git merge fran, estás trayendo fran a main.

9. Escenarios delicados de merge

ObjetivoSecuencia recomendadaComentario
Traer main a tu rama fran git checkout main
git pull origin main
git checkout fran
git merge main		 Este flujo actualiza primero main y luego integra esos cambios en tu rama de trabajo.
Fusionar fran en main git checkout main
git pull origin main
git merge fran
git push origin main		 Hazlo solo cuando main esté limpio y estés segura de que la rama ya fue probada.
Revisar antes de fusionar git fetch origin
git log --oneline --graph --all
git diff main..fran		 Muy útil antes de hacer un merge delicado en proyectos compartidos.
Merge con commit explícito git checkout main
git merge --no-ff fran		 Deja un hito visible en el historial. Útil para features importantes.
Antes de cualquier merge delicado:
  • Ejecuta git status y verifica que el árbol esté limpio.
  • Haz git stash si tienes trabajo a medio hacer.
  • Actualiza la rama base con git pull origin main.
  • Revisa diferencias con git diff o el historial con git log --graph --all.

13. Crear repositorio privado desde local

Idea general: puedes partir desde una carpeta local ya existente, inicializar Git, hacer el primer commit y luego publicar el proyecto como repositorio privado. Esto se puede hacer en distintas plataformas de alojamiento de código como GitHub, GitLab o Bitbucket. Dependiendo de la plataforma, puedes usar la interfaz web o herramientas de línea de comandos como gh (GitHub CLI) o equivalentes.
Paso / Comando ¿Para qué sirve? Sintaxis Ejemplo
git init Inicializa el repositorio local en la carpeta actual del proyecto. git init git init
git add . Agrega todos los archivos actuales al área de preparación (staging). git add . git add .
git commit -m Crea el primer commit del proyecto con un mensaje descriptivo. git commit -m "mensaje" git commit -m "Primer commit"
Crear repo privado en GitHub Desde la interfaz web de GitHub se crea un repositorio nuevo marcándolo como privado. Interfaz web Private repository
git branch -M main Renombra o fija la rama principal con el nombre main. git branch -M main git branch -M main
git remote add origin Conecta el repositorio local con el repositorio remoto privado. git remote add origin URL git remote add origin git@github.com:usuario/proyecto.git
git push -u origin main Sube el contenido local al remoto y deja configurada la rama de seguimiento. git push -u origin main git push -u origin main
Importante: Git por sí solo no crea repositorios privados en GitHub. En el flujo manual, primero debes crear el repositorio privado en la plataforma web y luego enlazarlo con git remote add origin. Si usas gh, puedes crear y publicar el repositorio privado directamente desde la terminal.
